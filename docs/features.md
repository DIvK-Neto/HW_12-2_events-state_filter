# Функциональность

## Основные возможности

- **Фильтрация портфолио** по категориям: "All", "Websites", "Business Cards", "Flayers".
- **Три компонента**:
  - `Portfolio` (классовый) – хранит состояние (активный фильтр) и управляет фильтрацией.
  - `Toolbar` (функциональный) – отображает кнопки-фильтры, получает список фильтров и функцию обратного вызова.
  - `ProjectList` (функциональный) – отображает список карточек проектов.
- **Визуальная обратная связь**: активный фильтр выделен синим фоном и белым текстом.
- **Адаптивная сетка**: на широких экранах 3 колонки, на планшетах 2, на мобильных 1.

## Технические детали

- Состояние `selectedFilter` хранится только в `Portfolio` и обновляется через `setState`.
- Фильтрация происходит в `getFilteredProjects()`: если фильтр "All" – возвращаются все проекты, иначе проекты, у которых `category === selectedFilter`.
- Компоненты не имеют собственного состояния (stateless), кроме главного.
- Данные передаются через пропсы, обратный вызов `onSelectFilter` передаётся из `Portfolio` в `Toolbar`.

## Взаимодействие компонентов

1. **Portfolio** получает массив проектов через пропс `projects`.
2. **Portfolio** вычисляет уникальные категории и формирует список фильтров (добавляя "All" в начале).
3. **Portfolio** передаёт в **Toolbar**:
   - `filters` – Set с категориями (преобразуется в массив внутри Toolbar)
   - `selected` – текущий выбранный фильтр
   - `onSelectFilter` – функцию для смены фильтра
4. При клике на кнопку в **Toolbar** вызывается `onSelectFilter(filter)`.
5. **Portfolio** обновляет `state.selectedFilter`, пересчитывает отфильтрованный список и передаёт его в **ProjectList** через пропс `projects`.
6. **ProjectList** отображает карточки проектов.

## Стилизация

- **Toolbar**: кнопки с отступами, активная кнопка – синий фон, белый текст.
- **ProjectList**: сетка на CSS `column-count` (адаптивность через медиазапросы). Каждая карточка содержит изображение на всю ширину.
